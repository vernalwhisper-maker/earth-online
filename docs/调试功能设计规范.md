# 调试功能模块设计规范

> 版本：v1.0
> 适用范围：地球Online 隐藏式开发者调试模式

---

## 零、核心铁律

以下规则为**最高优先级**，任何实现不得违反：

1. **不自动打包** — 未经用户明确指令，不得执行 `npm run build`、`gradlew assembleDebug` 或其他任何打包/构建/APK 生成操作
2. **不移位** — 调试效果不得改变组件原始位置（position、margin、flex/grid 布局）
3. **总控优先** — 总开关关闭时，副开关无条件不生效

## 一、架构总览

```
设置 → 更多设置
  └─ 连点 7 次 "更多设置" 标题
       └─ 高级调试卡片
            ├─ [总开关] 启用调试
            ├─ [副开关+进入] 表/类/夹调试
            ├─ [副开关+进入] 标签栏调试
            ├─ [副开关+进入] 导航栏调试
            └─ [副开关+进入] 新建/AI按钮调试
```

### 分层逻辑

```
总开关 (advancedDebug) —— 控制所有调试功能
  │
  └─ 副开关 (debugXxxEnabled) —— 控制单个模块
       │
       └─ 条件: advancedDebug && debugXxxEnabled
            │
            └─ 效果在首页生效
```

**规则**：副开关开启后，必须总开关也开启，调试效果才会在实际软件中生效。

## 二、状态管理规范

### 2.1 Store 字段命名

| 字段 | 类型 | 持久化 | 说明 |
|------|------|--------|------|
| `advancedDebug` | boolean | IndexedDB | 总开关 |
| `debugFABEnabled` | boolean | IndexedDB | 表/类/夹调试 |
| `debugTagBarEnabled` | boolean | IndexedDB | 标签栏调试 |
| `debugNavBarEnabled` | boolean | IndexedDB | 导航栏调试 |
| `debugFabGlassEnabled` | boolean | IndexedDB | 新建/AI按钮调试 |
| `devUnlocked` | boolean | 内存态 | 连点已触发（不持久化） |
| `devCardOpen` | boolean | 内存态 | 卡片显隐（不持久化） |

### 2.2 Store 方法规范

每个副开关需要三件套：

```js
// 1. 初始值
debugXxxEnabled: false,

// 2. loadSettings 读取
const debugXxxEnabled = (await getSetting("debugXxxEnabled")) ?? false;

// 3. set() 中写入
set({ ..., debugXxxEnabled });

// 4. setter
setDebugXxxEnabled: async (value) => {
  await setSetting("debugXxxEnabled", value);
  set({ debugXxxEnabled: value });
},

// 5. closeDevCard 联动
if (s.debugXxxEnabled) await setSetting("debugXxxEnabled", false);
set({ ..., debugXxxEnabled: false });
```

### 2.3 loadSettings 恢复入口

```js
devUnlocked: advancedDebug,
devCardOpen: advancedDebug,
```

当总开关已开启时，重新进入页面自动恢复卡片入口状态，避免"调试生效但入口消失"。

## 三、参数存储规范

### 3.1 独立键机制

每个调试模块使用独立的 localStorage 键，参数互不干扰：

```
earth-online-debug-segmented  → 表/类/夹调试
earth-online-debug-tagbar     → 标签栏调试
earth-online-debug-navbar     → 导航栏调试
earth-online-debug-fab        → 新建/AI按钮调试
```

### 3.2 默认值定义

在 `src/config/debugDefaults.js` 中：

```js
// 毛玻璃参数（新建/AI按钮、导航栏）
export const FAB_DEFAULTS = {
  blurPx: 18,
  saturation: 1.4,
  bgOpacity: 0.06,
  borderOpacity: 0.10,
  shadowOpacity: 0.5,
};

// LiquidGlass 参数（表/类/夹、标签栏）
export const DEBUG_DEFAULTS = {
  elasticity: 0.15,
  blurAmount: 0.0625,
  saturation: 140,
  displacementScale: 70,
  aberrationIntensity: 2,
  cornerRadius: 999,
  modeIdx: 0,
  overLight: false,
  shadowOpacity: 0.25,
};
```

### 3.3 CustomEvent 命名

```
earth-debug-segmented-changed
earth-debug-tagbar-changed
earth-debug-navbar-changed
earth-debug-fab-changed
```

## 四、调试页三级页面规范

### 4.1 文件结构

```
src/pages/subpages/
├── DebugPage.jsx          ← 表/类/夹调试
├── TagBarDebugPage.jsx    ← 标签栏调试
├── NavBarDebugPage.jsx    ← 导航栏调试
└── FABDebugPage.jsx       ← 新建/AI按钮调试
```

### 4.2 页面模板

```jsx
export default function XxxDebugPage({ onBack }) {
  const { darkMode } = useSettingsStore();

  // 1. 参数状态 — 从独立 localStorage 键读取
  const saved = loadParams();

  // 2. 每个参数一个 state
  const [param, setParam] = useState(saved?.param ?? DEFAULTS.param);

  // 3. 自动保存 + 通知首页实时生效
  useEffect(() => {
    const params = { param };
    saveParams(params);
    window.dispatchEvent(new CustomEvent("earth-debug-xxx-changed", { detail: params }));
  }, [param]);

  // 4. 恢复默认
  const handleReset = () => {
    localStorage.removeItem(STORAGE_KEY_XXX);
    setParam(DEFAULTS.param);
  };

  return (
    <motion.div ...>
      <button onClick={() => onBack?.("more")}>返回更多设置</button>
      <h1><Bug />调试标题</h1>

      {/* 实时预览 */}
      <section>
        <h2>实时预览</h2>
        <div style={{ background: `url("/测试专用.png")...` }}>
          {/* 预览内容 — 与实际效果 1:1 对齐 */}
        </div>
      </section>

      {/* 参数控制 */}
      <section>
        <div>
          <h2><Sliders />参数控制</h2>
          <button onClick={handleReset}><RotateCcw />恢复默认</button>
        </div>
        {/* RangeSlider / 开关 / 按钮 */}
      </section>
    </motion.div>
  );
}
```

### 4.3 返回导航

- 页面内返回按钮：`onBack?.("more")` → 回到更多设置二级页面
- Android 返回键：`settingsSubPage.startsWith("debug")` → 自动回到 `"more"`
- 跳转到调试页：`onBack?.("debug-xxx")` → SettingsPage 路由匹配

## 五、首页应用规范

### 5.1 条件判断

所有调试效果在首页的启用条件统一为：

```js
advancedDebug && debugXxxEnabled
```

在组件中引用：

```js
// App.jsx
const advancedDebug = useSettingsStore((s) => s.advancedDebug);
const debugFabGlass = useSettingsStore((s) => s.debugFabGlassEnabled) && advancedDebug;

// TabBar.jsx
const navDebug = useSettingsStore((s) => s.advancedDebug) && useSettingsStore((s) => s.debugNavBarEnabled);
```

### 5.2 参数读取 + 实时更新

```js
// 1. 初始化从 localStorage 读取
const [params, setParams] = useState(() => {
  try {
    const r = localStorage.getItem(STORAGE_KEY_XXX);
    return r ? JSON.parse(r) : DEFAULTS;
  } catch { return DEFAULTS; }
});

// 2. 监听 CustomEvent 实时更新
useEffect(() => {
  const h = (e) => setParams(e.detail);
  window.addEventListener("earth-debug-xxx-changed", h);
  return () => window.removeEventListener("earth-debug-xxx-changed", h);
}, []);
```

## 六、视觉效果规范

### 6.1 毛玻璃效果（新建/AI按钮、导航栏）

```css
background: rgba(255, 255, 255, 0.06);
backdrop-filter: blur(18px) saturate(1.4);
border: 1px solid rgba(255, 255, 255, 0.10);
box-shadow: 0 8px 32px rgba(0, 0, 0, 0.5),
            inset 0 1px 0 rgba(255, 255, 255, 0.12);
```

### 6.2 LiquidGlass 效果（表/类/夹、标签栏）

对不需要复杂自定义效果的区域使用 LiquidGlass 组件：

```jsx
<LiquidGlass
  cornerRadius={cornerRadius} padding="4px"
  elasticity={elasticity} blurAmount={blurAmount}
  saturation={saturation} displacementScale={displacementScale}
  aberrationIntensity={aberrationIntensity}
  mode={mode} overLight={overLight} shadowOpacity={shadowOpacity}
  wrapperStyle={{}}
>
  <div>内容</div>
</LiquidGlass>
```

### 6.3 禁止移位

- 调试效果不得改变组件原始位置
- 使用 `wrapperStyle={{}}` 让阴影层 absolute 定位脱离文档流
- 或使用外部 fixed 容器包裹纯 CSS 毛玻璃

## 八、命名与路由规范

### 8.1 路由

| 路由 ID | 对应页面 |
|---------|----------|
| `debug` | DebugPage（表/类/夹） |
| `debug-tagbar` | TagBarDebugPage |
| `debug-navbar` | NavBarDebugPage |

### 8.2 SettingsPage 路由注册

```jsx
if (settingsSubPage === "more") return <MoreSettingsPage onBack={(action) =>
  onSubPageChange?.(
    action === "debug" ? "debug" :
    action === "debug-tagbar" ? "debug-tagbar" :
    action === "debug-navbar" ? "debug-navbar" :

```

## 九、关闭行为规范

### 9.1 标题开关关闭

关闭调试卡片标题开关时，联动清除所有调试状态：

```js
closeDevCard: async () => {
  const s = get();
  // 重置所有持久化状态
  if (s.advancedDebug) await setSetting("advancedDebug", false);
  if (s.debugFABEnabled) await setSetting("debugFABEnabled", false);
  if (s.debugTagBarEnabled) await setSetting("debugTagBarEnabled", false);
  if (s.debugNavBarEnabled) await setSetting("debugNavBarEnabled", false);
  if (s.debugFabGlassEnabled) await setSetting("debugFabGlassEnabled", false);
  // 重置内存状态
  set({
    devCardOpen: false, devUnlocked: false,
    advancedDebug: false,
    debugFABEnabled: false, debugTagBarEnabled: false,
    debugNavBarEnabled: false, debugFabGlassEnabled: false,
  });
},
```

关闭后需重新连点 7 次才能让卡片重现。

## 十、设计原则

1. **独立性** — 每个调试模块参数独立存储、互不干扰
2. **实时性** — 参数变化通过 CustomEvent 即时同步到首页
3. **一致性** — 预览效果与首页应用效果完全 1:1
4. **零移位** — 调试效果不改变组件原始位置和布局
5. **可恢复** — 标题开关关闭时彻底清除所有调试状态
6. **总控优先** — 总开关关闭时，副开关无条件不生效
