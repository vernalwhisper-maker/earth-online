# 调试功能实现逻辑说明书

## 概述

v1.2.0 引入了隐藏式开发者调试模式，用于在软件中实时预览和调节 `liquid-glass-react` 液态玻璃组件的参数效果。该功能仿 Android 开发者模式，通过连续点击触发，不影响普通用户。

---

## 一、数据流

```
用户点击7次"更多设置"
  |
  v
MoreSettingsPage: devUnlocked=true, devCardOpen=true
  |
  +-> 高级调试卡片显示 (3个开关)
  |       |
  |       +-> 标题行开关: 控制卡片显隐 (devCardOpen)
  |       +-> 总开关(启用调试): 控制功能启用 (advancedDebug -> store)
  |       +-> 副开关(模糊调试): 控制具体功能 (debugFABEnabled -> store)
  |
  +-> "进入" -> SettingsPage.settingsSubPage="debug" -> DebugPage
  |       |
  |       +-> 参数滑块调参 -> localStorage 实时保存
  |
  +-> 返回首页
          |
          +-> App.jsx: advancedDebug && debugFABEnabled
                  |
                  +-> true: FAB 按钮用 <LiquidGlass> 包裹
                  +-> false: 使用原始 <motion.button>
```

## 二、核心状态

### settingsStore (persisted to IndexedDB)
```
advancedDebug: boolean     // 总开关 (控制所有调试功能)
debugFABEnabled: boolean   // 副开关 (控制模糊调试功能)
```

### MoreSettingsPage (component state, not persisted)
```
devUnlocked: boolean       // 7次连点已触发 (重置后需重新触发)
devCardOpen: boolean       // 卡片当前显隐
devTapCount: number        // 累计点击次数, 600ms 内
```

### DebugPage (localStorage)
```
earth-online-debug-params: {
  elasticity, blurAmount, saturation,
  displacementScale, aberrationIntensity,
  cornerRadius, modeIdx, overLight
}
```

## 三、关键实现逻辑

### 3.1 连点触发 (MoreSettingsPage.jsx)

```js
const handleTitleTap = useCallback(() => {
  setDevTapCount((prev) => prev + 1);
  if (devTimerRef.current) clearTimeout(devTimerRef.current);
  devTimerRef.current = setTimeout(() => {
    setDevTapCount(0);          // 600ms 后清零
    devTimerRef.current = null;
  }, 600);
}, []);

// 7次连点 -> 解锁 + 展示卡片
if (devTapCount >= 7 && !devUnlocked) {
  setDevTapCount(0);
  setDevUnlocked(true);
  setDevCardOpen(true);
}
```

### 3.2 标题行开关 (卡片显隐控制)

```js
const handleDevCardToggle = (open) => {
  setDevCardOpen(open);
  if (!open) setDevUnlocked(false);  // 关闭 -> 重置触发状态
};
```

### 3.3 总开关 + 副开关

```js
// 总开关: 控制是否应用所有调试功能
<GlassSwitch value={advancedDebug} onChange={async (v) => { await setAdvancedDebug(v); }} />

// 副开关: 控制是否应用模糊调试
<GlassSwitch value={debugFABEnabled} onChange={async (v) => { await setDebugFABEnabled(v); }} />
```

### 3.4 参数实时保存 (DebugPage.jsx)

```js
useEffect(() => {
  saveParams({ elasticity, blurAmount, saturation, ... });
}, [elasticity, blurAmount, saturation, ...]);
```

### 3.5 FAB 按钮条件渲染 (App.jsx)

```jsx
{advancedDebug && debugFABEnabled ? (
  // 调试模式: 用 LiquidGlass 包裹，保持固定位置
  <div className="fixed bottom-32 right-5 z-20">
    <LiquidGlass cornerRadius={999} padding="0" ...>
      <div className="w-14 h-14 bg-emerald rounded-full shadow-fab">
        <Plus size={24} className="text-white" />
      </div>
    </LiquidGlass>
  </div>
) : (
  // 普通模式: 原始 FAB 按钮，位置相同
  <motion.button className="fixed bottom-32 right-5 ...">...</motion.button>
)}
```

## 五、GlassSwitch prop 注意事项

GlassSwitch 组件接收 `value` 而非 `checked`:
```jsx
// 正确
<GlassSwitch value={myState} onChange={(v) => setMyState(v)} />

// 错误 (checked 会被忽略, 开关无反应)
<GlassSwitch checked={myState} onChange={(v) => setMyState(v)} />
```
